1. The Core Engine: State & Event

Unlike linear programming, LSL is a state machine. An object exists in one "State" (like 'On' or 'Off') and waits for an "Event" (like a touch) to trigger code.

The Golden Rule

Every script MUST have a state named default. This is the entry point where logic begins.

2. The Building Blocks

LSL uses strict typing. You must declare the container type before filling it. Misusing types (like putting text into an integer) causes syntax errors.

  • Integer: Whole numbers (32-bit).
  • Vector: 3D Coordinates <x,y,z>.
  • Key: Unique UUIDs for assets.

5. The Path to Mastery

LEVEL 1

The Color Changer

Learn the basics of touch_start and appearance functions. Click the object, change the color.

Master detection. Use sensor to find avatars nearby and llSay to talk to them.
